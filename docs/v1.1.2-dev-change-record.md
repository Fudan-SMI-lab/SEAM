## 相对 v1.1.1 / v1.1.1-dev 的主要新增能力

`v1.1.2-dev` 当前相对 `v1.1.1` / `v1.1.1-dev` 的主要新增能力包括：

- **通用 auto selector 候选替换**：`src/workflows/seam_auto_default.yaml` 将 MUSA/Muxi 候选从旧的长文件名实验 workflow 替换为更通用的 `musa_muxi_vllm.yaml`、`musa_muxi_sglang.yaml`、`musa_muxi_custom_op.yaml`、`musa_muxi_general.yaml`。
- **四个 MUSA/Muxi workflows**：新增 vLLM、SGLang、custom-op、general 四类 MUSA/Muxi workflow，保留 Muxi preset、container runtime、MetaX/MACA device/env 约束，并按场景区分 image、repair role 和 custom-op gate。
- **selector sidecar metadata**：workflow selector 在 materialized workflow 旁写入 `.selector.json`，记录 `selector_path`、`selected_path`、`materialized_path`，E2E 日志和 telemetry 可直接看到 selector 输入、实际选择和解析后 workflow。
- **auto image selected-first 行为**：当 auto image selector 选中某个候选 image 后，框架把该 image 放到候选列表第一位，再保留其他候选，避免后续 container create fallback 先尝试未选中的 image。
- **精确 container image 日志**：container create 日志改为输出 runtime、container name 和正在尝试的完整 image，便于区分多 image fallback 中到底是哪一个 image 失败或成功。
- **OpenCode server 诊断**：E2E 启动后记录非敏感 server diagnostics，包括 URL、来源、pid、cwd、配置文件是否存在和模型名列表，不输出 API key、token 或完整配置内容。
- **Phase 5 previous fixer output 隔离**：error analyzer history 中的 detailed fixer output 只展开最新一次有内容的 fixer 详情，避免旧修复的 `modified_files` 在后续分析中被误当成当前事实。
- **entry_script_action 迭代语义修复**：`entry_script_action` 只用于替换 Phase 3 `run_command`，不编辑 entry script 源文件；纯命令修订不消耗 repair iteration，有 repair role 时修订命令后仍会 dispatch 到对应 fixer。
- **最后一次 repair 后 validation-only 复跑**：当最后一次 repair 已执行但 loop state 中仍是旧失败退出码时，框架会做一次只跑验证 phases 的复跑，不再调用 analyzer、dispatch、fixer LLM phases，以获得修复后的真实 final gate 状态。
- **prompt 语义更新**：通用、container、MUSA、PPU error recovery prompts 都明确 `entry_script_action` 只替换 `run_command`，source edits 必须交给 repair agent，不能削弱 required reports、checks 或 custom-op evidence。
- **operator routing override 默认值调整**：custom-op operator routing override 从默认开启改为默认关闭，只有显式配置 `custom_op_operator_routing_override_enabled=true` 时才强制改路由。
- **回归测试补充**：新增或更新 workflow selector metadata、E2E selector log、auto image selected-first、container create logging、server diagnostics、entry script action 迭代、last-iteration validation-only rerun、operator routing override 等测试。
