# 贡献指南

SEAM 目前仍在持续迭代中。如果你热爱开源、愿意参与共建，无论是出于学习交流，或是有新的想法与优化思路，我们都十分欢迎你的加入。

## 参与方式
你可以通过以下多种形式为本项目贡献力量：
1. **反馈问题**：协助我们发现并修复代码缺陷
2. **功能提议**：提交新功能构想与优化建议
3. **代码贡献**：提交代码实现功能迭代与问题修复
4. **文档完善**：补充、修正项目相关文档
5. **项目推广**：向身边开发者分享、推荐本项目

### 反馈问题
若你在使用过程中发现 Bug 或异常问题，请按如下流程提交：
1. 前往 [Issues](https://github.com/Fudan-SMI-lab/SEAM/issues) 检索，确认该问题是否已有人提交
2. 若无相关记录，点击「New Issue」，填写清晰的标题与完整问题描述
3. 补充运行环境、复现步骤等关键信息，方便我们定位排查

### 功能提议
如果你有不错的新功能或优化想法，欢迎提出：
1. 先在 [Issues](https://github.com/Fudan-SMI-lab/SEAM/issues) 查看，避免重复提议
2. 确认无重复后，新建 Issue 并选择「Feature Request」模板详细描述方案
3. 可添加微信：Always031856（备注：nndeploy+姓名），进一步沟通想法

### 提交代码
如需贡献代码，请遵循以下流程：
1. Fork 当前仓库至个人账号
2. 基于 `main` 分支新建自定义分支（示例分支名：feature/xxx、fix/xxx）
3. 完成代码修改，严格遵循项目编码规范
4. 本地充分测试，确保改动可正常运行
5. 编写规范的提交说明，清晰描述变更内容
6. 将代码推送到你个人的 Fork 仓库
7. 向本仓库 `main` 分支发起 Pull Request

我们会及时审核 PR，与你沟通细节，共同完成代码合并。


# CONTRIBUTING

SEAM is actively under development. We welcome all open-source enthusiasts to join our community. Whether you want to learn or share ideas for improvement, your contributions are highly appreciated.

## Ways to Contribute
You can participate in the project in the following ways:
1. **Report Issues**: Help us identify and resolve bugs
2. **Feature Requests**: Share your ideas for new features and improvements
3. **Pull Requests**: Submit code changes to enhance the project
4. **Documentation**: Improve and enrich project documents
5. **Spread the Word**: Recommend and share this project with other developers

### Reporting Issues
If you encounter bugs or unexpected behaviors, please follow the steps below:
1. Check existing [Issues](https://github.com/Fudan-SMI-lab/SEAM/issues) to avoid duplicate reports.
2. Click "New Issue" and write a clear title and detailed description.
3. Include your operating environment and steps to reproduce the problem.

### Feature Requests
Got creative ideas to improve SEAM? Feel free to let us know:
1. Search [Issues](https://github.com/Fudan-SMI-lab/SEAM/issues) to check if the idea has been proposed before.
2. Create a new Issue with the "Feature Request" template to elaborate your plan.
3. Contact us via WeChat: Always031856 (Note: nndeploy + your name) for further discussion.

### Pull Requests
Please follow this workflow to submit code contributions:
1. Fork this repository.
2. Create a new branch based on the `main` branch for your changes.
3. Implement your modifications and comply with our coding standards.
4. Run sufficient local tests to guarantee code reliability.
5. Write concise and meaningful commit messages.
6. Push your commits to your forked repository.
7. Open a Pull Request targeting the `main` branch of this repository.

We will review your PR as soon as possible, communicate with you and merge qualified changes.
