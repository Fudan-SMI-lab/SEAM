## 相对 main 的新增能力

`v1.1.1-dev` 相对 `main` 的主要新增能力包括：

- **report-fixer/final-gate validator-runner 支持**：custom-op final gate 失败时可以通过独立 report fixer 修复最终报告，而不是让通用 fixer 猜测 validator 要求。
- **动态 repair role 描述**：analyzer prompt 使用 `{repair_role_descriptions}`，根据当前 workflow 可用角色动态注入，避免硬编码三角色。
- **available-role-aware analyzer/routing**：框架根据 workflow 中实际存在的 repair phase/route 控制 analyzer 可选角色，减少无效路由。
- **final-gate validator command/contract injection**：report fixer 可以看到真实可执行的 final-gate validator 命令和 contract 信息。
- **完整 final-gate 错误输出**：final-gate runner 保留 `error_count` 和完整错误列表，避免只给截断 summary 导致 agent 无法修复。
- **report-fixer prompt templates**：新增 generic/container/MUSA report-fixer prompts，用于最终报告和 final-gate schema 修复。
- **GLM-OCR 本地实验 workflows**：新增 `local_musa_muxi_vllm_glmocr_gpu5.yaml`、`local_musa_muxi_sglang_glmocr_gpu6.yaml` 和 `src/local_constraints_glmocr_vllm.md`，用于复现 vLLM/SGLang 实验。
- **测试覆盖**：新增或更新 repair role prompts、repair loop、workflow executor 相关测试，覆盖动态角色、report-fixer 路由和 validator-runner 行为。

## 复现实验注意事项

- 长 E2E 必须设置足够外层 timeout；GLM-OCR vLLM 本次耗时接近 3 小时，建议外层至少 4 小时以上。
- 为保证可比性，应使用与实验一致的 OpenCode server 配置；本次实验使用 `http://127.0.0.1:4097`，server 从 `/home/zihang/opencode_test/SEAM` 配置启动。
- 不应把 Deepwave、GLM-OCR、vLLM、SGLang 的个案修复硬编码进框架；框架层改动必须保持泛化。
- workflow 中 `cleanup: false` 或测试中保留输出时，容器和 output project 可能不会自动删除，复测前需人工确认和清理。
- 文中 MACA/MUSA 复现实验结果不等价于 PPU/NPU 可直接复现；PPU/NPU 需要对应 image、device、env 和 workflow 适配。
