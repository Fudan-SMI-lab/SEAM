Migration of MinerU 3.1.0 (document parsing tool) via vLLM serving route on Ascend NPU completed successfully after 3 repair iterations. One high-value operator-incompatibility experience emerged: vLLM's fused_moe module unconditionally imports CUDA-specific C++ extensions (fused_marlin_moe, fused_moe) that fail on Ascend NPU due to missing fused_topk CUDA symbol. The fix — wrapping these imports in try/except ImportError guards — is NPU-native, verified (FULL_PASS gate), generalizable to any vLLM+Ascend NPU deployment, and non-trivial (requires understanding the multi-hop import chain: fused_marlin_moe.py → fused_moe.py → _custom_ops → fused_moe_module C++ extension). Combined with vllm_ascend's existing patch_module.py dummy-module mechanism, this is a proven pattern worth capturing as a skill for future vLLM-on-Ascend migrations.
