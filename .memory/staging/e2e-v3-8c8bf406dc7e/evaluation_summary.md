The Deepwave FWI migration to Ascend NPU 910B2C (CANN 8.5.0) succeeded in its core objective: scalar FWI propagation, gradient descent, and loss reduction validated on NPU over 3 epochs (loss 0.006883→0.006252). Five distinct, non-trivial repair patterns were discovered: (1) Python import shadowing — `import deepwave.acoustic as _mod` resolved to the exported function `acoustic()` rather than the module, breaking NPU custom-op patching; fixed by switching to `importlib.import_module("deepwave.acoustic")`. (2) Custom compiled backends absent on NPU — `get_backend_function()` for CUDA-compiled .so files crashes; fixed with a `MISSING_COMPILED_BACKEND` early-return guard in `backend_utils.py`. (3) CANN TBE requires non-obvious Python packages (decorator, attrs, psutil, cffi, protobuf) that cause cryptic import failures when missing. (4) torchvision operators like `nms` have no NPU kernel; gracefully degraded LPIPS metric with informative skip rather than crash. (5) Multiple pybind .so files in the same project cause sys.path accumulation bugs when loaded via `sys.path.insert(0, ...)`; `importlib.util.spec_from_file_location()` by absolute path is the correct approach.
