MinerU vLLM serving path migrated to Ascend NPU (CANN 8.1.RC1) with FULL_PASS in 4 Phase 5 iterations. The migration was remarkably low-complexity: zero operator-level CUDA incompatibilities were found. The pre-built NPU-aware vLLM at /vllm-workspace/vllm/ handled all hardware abstraction. The only issue — a port mismatch between the validation script (DEFAULT_SERVING_PORT=8000) and the actual server (hardcoded port 30000) — was a configuration validation artifact, not a CUDA-to-NPU adaptation. No experiences from the memory system were applicable (all experience arrays empty). The key strategic insight is that vLLM-based serving routes with NPU-aware vLLM builds require minimal operator-level migration effort, making them a distinct low-complexity category.
