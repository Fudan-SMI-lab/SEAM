# SEAM Experience Memory

`.memory/` is the generated self-evolution workspace for SEAM migrations. It is written by the Phase 7 experience pipeline and read by later runs when they need reusable migration knowledge.

## Layout

- `.memory/memory/` stores the experience registry: indexes, case studies, staging candidates, refined lessons, promotion proposals, archives, and quarantine records.
- `.memory/skills/` stores generated/promoted runtime skills produced from validated migration experience. These skills can be referenced by workflow `runtime_skills` and injected into future OpenCode migration roles.
- `.skills/` at the repository root is separate. It is the pre-run skill-pack location for users and developers and is not generated by the experience system.

## Flow

1. A migration run writes phase outputs and validation evidence.
2. Phase 7 evaluates useful failures or successes and records candidates under `.memory/memory/staging/`.
3. Refined experiences are cataloged in `.memory/memory/index/experiences.jsonl` and summarized in `.memory/memory/manifest.json`.
4. Repeated or high-confidence lessons are promoted into `.memory/skills/<skill-name>/`.
5. Later runs resolve explicit workflow skills from `.memory/skills/` first and user/developer skills from `.skills/` as local overrides or additions.

Generated run reports such as `e2e-reports/` and migrated project copies under `output_projects/` are not part of this memory store and should not be committed.
