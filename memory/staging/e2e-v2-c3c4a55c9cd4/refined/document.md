# torch-npu 2.5.1 NumPy 2.x incompatibility — np.float_ removal breaks runtime

When creating a fresh virtual environment and installing torch-npu 2.5.1, pip resolves numpy to the latest available version (2.2.6 at time of discovery). NumPy 2.0 removed the np.float_ alias in favor of np.float64, but torch-npu 2.5.1's internal code still references np.float_, causing an AttributeError at runtime. This is a silent dependency mismatch because torch-npu's pip metadata does not pin an upper bound on numpy. The fix is to explicitly downgrade numpy to 1.26.x, which is the latest 1.x series confirmed to work with torch-npu 2.5.1. This issue is particularly insidious because pip's dependency resolver does not flag it — numpy installs successfully, and the error only manifests when torch-npu internals attempt to use np.float_ during tensor operations or memory management. Pre-pinning numpy<2.0 in requirements.txt or installing it before torch-npu prevents the issue entirely. This pattern will repeat for any torch-npu version built against NumPy 1.x APIs until upstream patches the compatibility layer.
